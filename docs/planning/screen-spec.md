# MVP2 화면 정의서

업데이트: 2026-03-05

## 공통 규칙

- 반응형 기준: Desktop 1440 / Tablet 1024 / Mobile 390
- 인증 필요 화면은 Access Token 기준으로 진입 제어
- 오류는 공통 에러 컴포넌트로 표시

## 1) 로그인/온보딩 화면

### 목적

- 카카오 로그인 진입 및 신규 사용자 온보딩 완료

### 진입 조건

- 토큰 없음
- 또는 `SIGNUP_REQUIRED` 상태

### UI 구성 요소

- 로그인 게이트 카드
- 카카오 로그인 버튼
- 온보딩 입력(닉네임/동의)

### 사용자 액션

- 카카오 로그인 클릭
- 온보딩 완료 제출

### 상태 전이

- 성공: 채팅 메인 진입
- 실패: 에러 메시지 표시

### API/소켓 연동

- `GET /auth/kakao/callback`
- `POST /auth/signup/complete`
- `POST /auth/refresh`

### 예외/오류

- invalid code
- signup token 만료
- refresh 실패 시 로그아웃

## 2) 방 목록 화면

### 목적

- 참여 방 탐색/검색/생성

### 진입 조건

- 로그인 성공

### UI 구성 요소

- 검색 입력
- 방 목록 리스트
- FAB(+) + 방 생성 모달
- 우측 상단 사용자 메뉴(플로팅)

### 사용자 액션

- 방 검색
- 방 클릭(입장)
- 방 생성

### 상태 전이

- 방 생성 성공 시 자동 입장
- 검색 결과 없음 안내

### API/소켓 연동

- `GET /api/chatrooms`
- `POST /api/chatrooms`
- `join_room`

### 예외/오류

- 생성 실패
- 목록 조회 실패

## 3) 채팅 상세 화면

### 목적

- 메시지 송수신/히스토리 조회/참여자 확인

### 진입 조건

- 방 입장 완료(`room_joined`)

### UI 구성 요소

- 채팅 헤더 + 참여자 메뉴
- 메시지 리스트
- 입력창 + 전송 버튼
- 모바일 뒤로가기(←)

### 사용자 액션

- 메시지 전송(Enter/버튼)
- 참여자 드롭다운 확인
- 모바일에서 목록 복귀

### 상태 전이

- 메시지 수신 시 하단 오토스크롤
- 방 변경 시 히스토리 재조회

### API/소켓 연동

- `GET /api/chatrooms/:id/messages`
- `send_message`
- `receive_message`
- `room_participants`

### 예외/오류

- 히스토리 조회 실패
- 권한 없음(forbidden)

## 4) 사용자 메뉴/설정 화면

### 목적

- 내 정보 확인, 닉네임/프로필 설정, 로그아웃

### 진입 조건

- 로그인 상태

### UI 구성 요소

- 플로팅 점3개 메뉴
- 내 정보 모달
- 설정 모달

### 사용자 액션

- 내 정보 열기
- 닉네임 수정 저장
- 로그아웃

### API/소켓 연동

- `PATCH /auth/profile`
- 소켓 disconnect

### 예외/오류

- 닉네임 검증 실패
- 저장 실패

## 5) 오류/빈 상태 화면

### 목적

- 오류 원인을 사용자가 인지하고 복구 가능하게 안내

### 진입 조건

- API/소켓 오류
- 데이터 없음

### UI 구성 요소

- 공통 오류 텍스트
- 빈 상태 텍스트

### 사용자 액션

- 재시도/다시 입장

### 상태 전이

- 재시도 성공 시 정상 화면 복귀

### API/소켓 연동

- 실패한 API 재호출
- 소켓 재연결

### 예외/오류

- 인증 만료
- DB 연결 실패
