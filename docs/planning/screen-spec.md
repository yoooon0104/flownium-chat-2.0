# MVP2 화면 정의서

업데이트: 2026-03-12

## 공통 규칙

- 반응형 기준: Desktop 1440 / Tablet 1024 / Mobile 390
- 실제 모바일/태블릿 전환 기준은 현재 구현상 `1023px 이하`
- 인증 필요 화면은 Access Token 기준으로 진입 제어
- 모바일과 데스크톱은 정보 구조는 같고, 내비게이션 방식만 다르게 간다

## 1) 로그인/온보딩 화면

### 목적

- 카카오 로그인 진입
- 신규 사용자 온보딩 완료
- 이메일 회원가입/인증/로그인
- 이메일 비밀번호 재설정

### UI 구성 요소

- 브랜드 로고
- 카카오 로그인 버튼
- 로그인 화면 진입 카드
- 회원가입 화면 진입 카드
- 이메일 로그인 폼
- 이메일 회원가입 폼(닉네임, 비밀번호, 비밀번호 확인)
- 서비스 이용 약관 요약 카드
- 회원가입 약관 동의 체크박스(필수)
- 비밀번호 재설정 화면 진입 카드
- 이메일/비밀번호 입력
- 회원가입 시 닉네임 입력
- 인증 코드 입력
- 온보딩 입력(닉네임/동의)
- 향후 기본 회원가입 진입 버튼(계획)
- 향후 비회원 이용 진입 버튼(계획)

### 상태 전이

- 성공: 메인 화면 진입
- 이메일 회원가입 시작: 인증 코드 입력 단계 진입
- 이메일 회원가입 완료: 메인 화면 진입 후 카카오 간편로그인 연결 유도 모달 표시 가능
- 실패: 에러 메시지 표시

### 연동 포인트

- `GET /auth/kakao/callback`
- `POST /auth/signup/complete`
- `POST /auth/email/signup/start`
- `POST /auth/email/signup/verify`
- `POST /auth/email/login`
- `POST /auth/email/password-reset/start`
- `POST /auth/email/password-reset/verify`
- `POST /auth/refresh`

### 현재 규칙 / 향후 확장 메모

- 카카오 로그인은 간편로그인 수단으로 동작
- 이메일 회원가입은 인증 코드 검증 후 즉시 가입 완료 + 로그인 성공
- 비밀번호 재설정은 인증 코드 검증 후 새 비밀번호를 저장하고 로그인 화면으로 복귀
- 개발 단계에서는 실제 메일 발송 대신 서버 로그/DB로 인증 코드를 확인
- 카카오와 이메일 계정은 자동 병합하지 않음
- 이메일 회원은 내 정보 화면에서 카카오 계정 연결을 시작할 수 있음
- 카카오 연결 해제는 다른 로그인 수단이 남아 있을 때만 내 정보 화면에서 허용
- 회원가입은 로그인 화면 안의 탭 전환이 아니라 별도 회원가입 화면처럼 분리해 보여준다
- 회원가입은 약관 동의, 이메일 형식, 닉네임 2~20자, 비밀번호 확인 일치, 영문/숫자 포함 비밀번호 규칙을 따른다
- 비회원 이용은 익명방 전용 진입으로 분리할 계획

## 2) 메인 리스트 화면

### 목적

- 친구 목록, 방 목록, 알림 허브를 진입점으로 제공

### 데스크톱 UI

- 좌측 패널
  - 프로필 헤더
  - 검색 버튼
  - 친구 추가 버튼
  - 알림 버튼
  - 메뉴 버튼
  - `Friends | Rooms` 탭
  - 현재 탭의 리스트 본문
- 우측 패널
  - 빈 상태 또는 채팅 상세

### 모바일 UI

- 상단 헤더 최소화
- 하단 탭바
  - 친구
  - 채팅방
  - 알림
  - 설정
- 탭 전환 시 별도 화면처럼 이동

### Friends 탭

- 수락된 친구 목록만 표시
- 닉네임/이메일/프로필 이미지 표시
- 데스크톱: 더블 클릭으로 1:1 시작
- 모바일: 액션 시트 후 채팅 시작
- 탈퇴 회원은 `(탈퇴한 회원)` suffix와 muted row로 표시
- 탈퇴 회원은 프로필/1:1 시작 진입을 막음

### Rooms 탭

- 방 목록 표시
- 방 이름
- 최근 메시지
- 최근 시각
- unread badge

### 연동 포인트

- `GET /api/friends`
- `GET /api/chatrooms`
- `GET /api/notifications`

## 3) 친구 추가 모달

### 목적

- 이메일로 사용자 검색 후 친구 요청 생성

### UI 구성 요소

- 검색 입력(이메일)
- 검색 결과 리스트
- 친구 요청 버튼

### 상태 전이

- 요청 성공 시 알림/친구 목록 재조회
- 이미 요청/친구 상태면 버튼 비활성 또는 상태 표시

### 연동 포인트

- `GET /api/friends/search`
- `POST /api/friends/request`

## 4) 채팅 생성 모달

### 목적

- 친구 선택 기반으로 1:1 또는 그룹 채팅 생성

### UI 구성 요소

- 친구 선택 리스트
- 선택 상태 표시
- 그룹 이름 입력(2명 이상 선택 시, 선택)
- 탈퇴 회원은 선택 리스트에서 제외

### 상태 전이

- 1명 선택:
  - 기존 2인 방 재사용 또는 새 방 생성
- 2명 이상 선택:
  - 그룹방 생성 후 자동 입장
  - 이름을 비우면 현재 선택된 멤버 닉네임 기반 기본 이름 생성

### 연동 포인트

- `POST /api/chatrooms`

## 5) 알림 허브

### 데스크톱

- 벨 버튼 클릭 시 드롭다운

### 모바일

- 알림 전용 화면으로 이동

### UI 구성 요소

- 받은 친구 요청
- 최근 알림 리스트
- unread count 배지

### 현재 규칙

- 받은 요청만 처리 대상
- 보낸 요청은 별도 섹션으로 두지 않음
- 최근 알림은 허브 열람 시 자동 읽음 처리
- 알림 허브는 현재 `friend_request` 중심으로 유지하고, 방 초대는 별도 알림 카드로 노출하지 않음

### 연동 포인트

- `GET /api/notifications`
- `PATCH /api/notifications/:id/read`
- `notification_created`
- `notification_read`
- `friendship_updated`

## 6) 채팅 상세 화면

### 목적

- 메시지 송수신
- unread 상태 표시
- 참여자 확인

### UI 구성 요소

- 채팅 헤더
- 참여자 버튼 및 목록
- 참여자 드롭다운 내 친구 초대 버튼
- 참여자 드롭다운 내 나가기 버튼
- 이전 메시지 더보기 버튼
- 메시지 리스트
- 메시지 옆 unread 숫자
- 시스템 메시지(초대/나가기) 구분 렌더링
- 입력창 + 전송 버튼
- 모바일 뒤로가기
- direct 상대가 탈퇴한 경우 읽기 전용 안내 + 입력 disabled

### 현재 규칙

- 내가 보낸 메시지에도 optimistic append 적용
- 서버 응답 시 `clientMessageId`로 실제 메시지로 치환
- 시간 표시는 `시:분`까지만 사용
- 현재 방 진입 시 읽음 처리
- 상대 메시지 수신 시 현재 방이면 즉시 읽음 처리
- 이전 메시지는 커서 기반으로 추가 로드
- 읽음 처리 후 기존 메시지 unread 숫자는 실시간 `message_updated`로 보정
- 1:1 방에서 친구 초대 시 기존 방을 유지하고 새 다인방으로 자동 입장
- 다인방에서 친구 초대 시 현재 방에 즉시 멤버 추가
- 1:1 방은 한 명이 나가면 삭제
- 다인방은 멤버가 0명이 될 때만 삭제
- 초대/나가기 이벤트는 시스템 메시지로 표시
- direct 상대가 탈퇴한 경우 `탈퇴한 회원입니다.` 안내를 보여주고 메시지 전송을 막음

### 연동 포인트

- `GET /api/chatrooms/:id/messages`
- `POST /api/chatrooms/:id/invite`
- `POST /api/chatrooms/:id/leave`
- `PATCH /api/chatrooms/:id/read`
- `send_message`
- `receive_message`
- `message_updated`
- `room_joined`
- `room_participants`
- `room_updated`
- `room_deleted`

## 7) 설정/내 정보 화면

### 현재 상태

- 데스크톱은 메뉴/모달 구조
- 모바일은 프로필 진입과 하단 탭을 통해 `내 정보`와 `설정` 화면을 각각 연다
- 우측 상단 메뉴 안에 `내 정보`, `설정`, `로그아웃` 항목을 둔다
- `내 정보`는 닉네임 변경, 이메일 확인, 비밀번호 변경, 카카오 계정 연결/해제, 회원탈퇴를 담당한다
- `설정`은 테마(light / dark / system) 같은 앱 환경 설정만 담당한다

### 현재 규칙

- `내 정보`의 기본 정보는 read-only로 보이고, `내 정보 변경`을 눌렀을 때 edit mode로 전환된다
- edit mode에서는 `확인 / 취소`로 저장 여부를 명시적으로 결정한다
- 회원탈퇴 실행 전 브라우저 confirm으로 한 번 더 확인
- 이미 연결된 카카오 계정은 내 정보 화면에서 `연결됨` 상태로만 표시
- 카카오 계정 연결은 내 정보 화면에서 시작하고, 카카오 OAuth 완료 후 현재 로그인한 계정에 귀속된다
- 카카오 계정 연결 해제는 마지막 로그인 수단이 아닐 때만 가능하다
- 비밀번호 변경은 이메일 identity가 연결된 계정만 가능하다
- 이메일 변경은 이메일 identity가 연결된 계정만 가능하며, 현재 비밀번호 확인과 인증 코드 검증을 거친다
- 회원탈퇴 성공 시 세션을 비우고 로그인 게이트로 복귀
- direct 방은 tombstone 상태로 유지되고, group 방은 남은 멤버 기준으로 유지/삭제 규칙을 따른다
- 데스크톱 모달과 모바일 내 정보 화면은 눈에 띄지 않는 subtle scrollbar를 사용한다

### 연동 포인트

- `PATCH /auth/profile`
- `PATCH /auth/password`
- `POST /auth/email/change/start`
- `POST /auth/email/change/verify`
- `POST /auth/kakao/link/start`
- `GET /auth/kakao/callback`
- `DELETE /auth/account`

## 8) 오류/빈 상태

### 목적

- 사용자가 현재 상황을 바로 이해하고 복구 경로를 알 수 있게 표시

### 현재 예시

- 친구 없음
- 방 없음
- 알림 없음
- 인증 오류
- 네트워크/DB 오류

## 9) 예정 화면: 기본 회원가입 / 인증 수단 분리

### 목적

- 카카오 간편로그인 외에도 기본 회원가입을 지원하기 위한 화면 확장

### 계획 메모

- 카카오는 로그인 수단 중 하나로 위치 조정 완료, 다음 단계는 기본 회원가입 연결
- 기본 회원가입은 최소 서비스 정보 기준으로 가입
- `User` 본체와 로그인 수단(`AuthIdentity`)을 분리하는 구조를 기준으로 확장

## 10) 예정 화면: 익명방 / 비회원 이용

### 목적

- 비회원 또는 회원이 익명방에 입장할 수 있는 별도 채팅 경험 제공

### 계획 메모

- 로그인 화면에서 `비회원 이용` 선택 가능
- 비회원은 닉네임 설정 후 익명방만 사용 가능
- 익명방은 방 토큰 URL과 비밀번호로 입장
- 회원도 익명방에 들어갈 수 있지만 방별 익명 닉네임을 사용
- 회원의 채팅 목록에서는 익명방을 일반 채팅과 구분해 표시
