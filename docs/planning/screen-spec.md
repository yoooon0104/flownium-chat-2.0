# MVP2 화면 정의서

업데이트: 2026-03-10

## 공통 규칙

- 반응형 기준: Desktop 1440 / Tablet 1024 / Mobile 390
- 실제 모바일/태블릿 전환 기준은 현재 구현상 `1023px 이하`
- 인증 필요 화면은 Access Token 기준으로 진입 제어
- 모바일과 데스크톱은 정보 구조는 같고, 내비게이션 방식만 다르게 간다

## 1) 로그인/온보딩 화면

### 목적

- 카카오 로그인 진입
- 신규 사용자 온보딩 완료

### UI 구성 요소

- 브랜드 로고
- 카카오 로그인 버튼
- 온보딩 입력(닉네임/동의)

### 상태 전이

- 성공: 메인 화면 진입
- 실패: 에러 메시지 표시

### 연동 포인트

- `GET /auth/kakao/callback`
- `POST /auth/signup/complete`
- `POST /auth/refresh`

## 2) 메인 리스트 화면

### 목적

- 친구 목록, 방 목록, 알림 허브를 진입점으로 제공

### 데스크톱 UI

- 좌측 패널
  - 프로필 헤더
  - 검색 버튼
  - 친구 추가 버튼
  - 알림 버튼
  - 설정 버튼
  - `Friends | Rooms` 탭
  - 현재 탭의 리스트 본문
- 우측 패널
  - 빈 상태 또는 채팅 상세

### 모바일 UI

- 상단 헤더 최소화
- 하단 탭바
  - 친구
  - 채팅방
  - 알림
  - 설정
- 탭 전환 시 별도 화면처럼 이동

### Friends 탭

- 수락된 친구 목록만 표시
- 닉네임/이메일/프로필 이미지 표시
- 데스크톱: 더블 클릭으로 1:1 시작
- 모바일: 액션 시트 후 채팅 시작

### Rooms 탭

- 방 목록 표시
- 방 이름
- 최근 메시지
- 최근 시각
- unread badge

### 연동 포인트

- `GET /api/friends`
- `GET /api/chatrooms`
- `GET /api/notifications`

## 3) 친구 추가 모달

### 목적

- 이메일/닉네임으로 사용자 검색 후 친구 요청 생성

### UI 구성 요소

- 검색 입력
- 검색 결과 리스트
- 친구 요청 버튼

### 상태 전이

- 요청 성공 시 알림/친구 목록 재조회
- 이미 요청/친구 상태면 버튼 비활성 또는 상태 표시

### 연동 포인트

- `GET /api/friends/search`
- `POST /api/friends/request`

## 4) 채팅 생성 모달

### 목적

- 친구 선택 기반으로 1:1 또는 그룹 채팅 생성

### UI 구성 요소

- 친구 선택 리스트
- 선택 상태 표시
- 그룹 이름 입력(2명 이상 선택 시)

### 상태 전이

- 1명 선택:
  - 기존 2인 방 재사용 또는 새 방 생성
- 2명 이상 선택:
  - 그룹방 생성 후 자동 입장

### 연동 포인트

- `POST /api/chatrooms`

## 5) 알림 허브

### 데스크톱

- 벨 버튼 클릭 시 드롭다운

### 모바일

- 알림 전용 화면으로 이동

### UI 구성 요소

- 받은 친구 요청
- 최근 알림 리스트
- unread count 배지

### 현재 규칙

- 받은 요청만 처리 대상
- 보낸 요청은 별도 섹션으로 두지 않음
- 최근 알림은 허브 열람 시 자동 읽음 처리

### 연동 포인트

- `GET /api/notifications`
- `PATCH /api/notifications/:id/read`
- `notification_created`
- `notification_read`
- `friendship_updated`

## 6) 채팅 상세 화면

### 목적

- 메시지 송수신
- unread 상태 표시
- 참여자 확인

### UI 구성 요소

- 채팅 헤더
- 참여자 버튼 및 목록
- 메시지 리스트
- 메시지 옆 unread 숫자
- 입력창 + 전송 버튼
- 모바일 뒤로가기

### 현재 규칙

- 내가 보낸 메시지에도 optimistic append 적용
- 서버 응답 시 `clientMessageId`로 실제 메시지로 치환
- 시간 표시는 `시:분`까지만 사용
- 현재 방 진입 시 읽음 처리
- 상대 메시지 수신 시 현재 방이면 즉시 읽음 처리

### 연동 포인트

- `GET /api/chatrooms/:id/messages`
- `PATCH /api/chatrooms/:id/read`
- `send_message`
- `receive_message`
- `room_joined`
- `room_participants`

## 7) 설정/내 정보 화면

### 현재 상태

- 데스크톱은 메뉴/모달 구조
- 모바일은 하단 탭을 통해 접근

### 확장 예정

- 알림 화면처럼 별도 설정 화면으로 분리 가능
- 다크모드, 언어, 로그아웃, 회원탈퇴 확장 예정

## 8) 오류/빈 상태

### 목적

- 사용자가 현재 상황을 바로 이해하고 복구 경로를 알 수 있게 표시

### 현재 예시

- 친구 없음
- 방 없음
- 알림 없음
- 인증 오류
- 네트워크/DB 오류
