# MVP2~MVP3 요구사항 정의서

업데이트: 2026-03-09

## 0. MVP1 기준 상태

MVP2와 MVP3 확장은 아래 MVP1 완료 상태를 기준으로 진행한다.

1. 카카오 OAuth + JWT 인증
2. 그룹 채팅방 생성/목록/입장
3. 실시간 메시지 송수신 및 메시지 이력 조회
4. 참여자 online/offline 표시
5. 방 검색 + FAB 기반 생성 모달
6. 로그인 게이트와 사용자 메뉴
7. 브랜딩 1차 적용
8. 운영 배포 1차와 운영 카카오 로그인 성공

## 1. 문서 목적

본 문서는 MVP2와 MVP3에서 어떤 기능을 어떤 우선순위로 확장할지 정의한다. 구현 전 범위 합의, 이후 기능 정의서/API 명세/WBS와의 기준 문서 역할을 한다.

## 2. 비용 정책

1. 포함: 무료 또는 자체 구현 가능한 기능
2. 제외: 유료 API, 유료 SaaS 의존 기능
3. 보류: 무료 플랜은 가능하지만 즉시 도입하지 않을 외부 서비스

## 3. 단계별 범위

### 3.1 MVP2-A 운영/기반

#### 목표
- 운영 배포와 로그인 흐름 안정화
- 오류 응답, 환경변수, 로그 기준 정리

#### 현재 상태
- Render/Vercel/Atlas 1차 셋업 완료
- 운영 카카오 로그인 성공
- redirect URI 슬래시 이슈 해결
- 운영 2탭 E2E만 잔여

#### In Scope
1. 운영 배포 셋업
2. `VITE_API_BASE_URL` 전환
3. 카카오 Redirect URI 운영/로컬 검증
4. `error.code` 기반 오류 표준화
5. 운영 로그/배포 런북 문서화
6. 브랜딩 1차 적용

#### Out of Scope
- 친구 도메인 UI/UX 확장
- `leave_room`
- 익명 채팅
- 회원탈퇴

### 3.2 MVP2-B 친구 기반 일반 채팅

#### 목표
- 친구 관계를 기반으로 1:1/그룹 채팅 생성 흐름 확립
- Friends/Rooms 중심 메인 UX 전환

#### 확정 정책
1. 메인 좌측 패널은 `Friends | Rooms` 2탭 구조
2. 기본 탭은 `Friends`
3. 검색은 상단 검색 버튼으로 열고, 현재 탭 기준으로 동작
4. Friends 탭은 수락된 친구 목록만 표시
5. 친구 요청/방 초대는 알림 허브에서 처리
6. 데스크톱에서는 친구 더블 클릭으로 1:1 시작
7. 모바일에서는 친구 탭 -> 액션 시트 -> 채팅 시작
8. `+` 버튼은 일반 채팅 생성 전용
9. 1명 선택 시 기존 2인 방 재사용 또는 새 생성
10. 2명 이상 선택 시 그룹 이름 입력 후 새 방 생성
11. 기존 1:1 방을 그룹으로 변형하지 않고 새 그룹방 생성
12. `leave_room`, 회원탈퇴, 익명 채팅은 이번 범위 제외

#### In Scope
1. 친구 검색
- 이메일, 닉네임 검색 지원
2. 친구 요청/처리
- 요청, 수락, 거절, 차단
3. 친구 요청 알림
4. Friends/Rooms UI
5. 친구 선택 기반 1:1/그룹 채팅 생성
6. 그룹 초대 알림
7. 프로필 이미지 표시 확장
- Friends 목록
- 친구 선택 모달
- 내 정보 모달 유지

#### Out of Scope
- `leave_room`
- 회원탈퇴
- 익명 채팅방
- 프로필 이미지 수정 업로드
- 다크모드/다국어

### 3.3 MVP3 무상 특화 기능

#### 목표
- 사용자 체감 기능 고도화

#### 후보 범위
1. 인앱 알림 UX 고도화
2. 메시지 검색/북마크/읽음 상태
3. 다국어(i18n)
4. 다크모드
5. 익명 채팅방
- 일반 채팅과 별도 탭
- 방 키 초대
- 방별 익명 닉네임
6. 회원탈퇴 및 계정 라이프사이클

## 4. MVP2-B 상세 요구사항

### FR-B01 친구 검색
- 이메일과 닉네임 기준 검색 지원
- 자기 자신은 검색 결과에서 제외
- 결과에는 현재 친구 상태 정보 포함

### FR-B02 친구 요청
- 친구 요청 생성 가능
- 동일 사용자쌍 중복 요청 차단
- 친구 요청 생성 시 상대에게 알림 생성

### FR-B03 친구 요청 처리
- 수락, 거절, 차단 지원
- 수락된 관계만 채팅 생성 대상

### FR-B04 Friends 탭
- Friends 탭은 친구 목록만 표시
- 친구 요청/방 초대는 벨 알림 허브로 분리
- 친구 목록은 닉네임/이메일 기준 정렬

### FR-B05 1:1 채팅 시작
- 데스크톱: 친구 더블 클릭
- 모바일: 친구 탭 -> 액션 시트 -> 채팅 시작
- 기존 2인 방이 있으면 재사용

### FR-B06 그룹 채팅 생성
- `+` 버튼 클릭 시 친구 선택 모달 진입
- 1명 선택 시 1:1 생성 또는 재사용
- 2명 이상 선택 시 그룹 이름 입력 후 새 생성
- 초대 대상에게 `room_invite` 알림 생성

### FR-B07 검색 UX
- 검색창은 상단 액션의 검색 버튼으로 열고 닫음
- Friends 탭에서는 친구 이름/이메일 검색
- Rooms 탭에서는 방 이름/최근 메시지 검색

### FR-B08 알림 허브
- 받은 친구 요청
- 보낸 친구 요청
- 최근 알림 목록
- 읽음 처리

## 5. 수락 기준(DoD)

### MVP2-A
1. 운영 도메인 카카오 로그인 성공
2. 방 생성/입장/메시지/참여자 흐름 정상
3. CORS/소켓 재연결 루프 문제 없음
4. 배포 런북으로 재현 가능

### MVP2-B
1. 친구 검색과 친구 요청/수락 흐름 정상
2. Friends/Rooms 탭이 의도한 구조로 동작
3. 1:1 방 재사용 규칙 정상
4. 그룹 채팅 생성과 초대 알림 정상
5. 알림 허브에서 요청/초대/읽음 처리 가능
