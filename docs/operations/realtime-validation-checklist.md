# 실계정 검증 체크리스트

업데이트: 2026-03-10

## 목적

- 실제 2계정 기준으로 친구, 채팅, 알림, unread, 참여자 상태가 새로고침 없이 정상 동작하는지 검증한다.
- 로컬 또는 운영 환경에서 동일한 순서로 반복 검증할 수 있도록 체크리스트를 고정한다.

## 사전 준비

- 계정 A, 계정 B 2개 준비
- 브라우저 2개 또는 일반창/시크릿창 분리
- 같은 환경에서 테스트
  - 로컬: `http://localhost:5173`
  - 운영: 배포 URL
- 서버와 프론트가 모두 최신 상태인지 확인

## 1. 로그인/세션

- [ ] 계정 A 로그인 성공
- [ ] 계정 B 로그인 성공
- [ ] 두 계정 모두 Friends 탭 진입 확인
- [ ] 새로고침 후 세션 유지 확인

## 2. 친구 요청/수락/거절

- [ ] 계정 A가 계정 B를 이메일 또는 닉네임으로 검색 가능
- [ ] 계정 A가 친구 요청 전송 가능
- [ ] 계정 B 알림 허브 또는 알림 화면에 새로고침 없이 친구 요청 반영
- [ ] 계정 B가 친구 요청 수락 가능
- [ ] 계정 A 화면에도 새로고침 없이 친구 상태 반영
- [ ] 계정 B가 친구 요청 거절 가능
- [ ] 거절 시 양쪽 화면 상태가 새로고침 없이 반영

## 3. 1:1 채팅

- [ ] Friends 탭에서 1:1 채팅 시작 가능
- [ ] 같은 친구로 다시 시작 시 기존 방 재사용
- [ ] 계정 A가 보낸 메시지가 계정 B 화면에 새로고침 없이 반영
- [ ] 계정 B가 다른 방을 보고 있을 때 Rooms 목록 unread 증가 확인
- [ ] 계정 B가 해당 방에 입장하면 unread 감소 확인

## 4. 그룹 채팅

- [ ] 계정 A가 + 버튼으로 친구 여러 명 선택 가능
- [ ] 그룹 이름 입력 후 새 방 생성 가능
- [ ] 초대 대상 알림 허브에 `room_invite` 반영
- [ ] 초대 대상이 방 입장 가능
- [ ] 방 헤더 참여자 수/목록이 입장 직후 표시

## 5. 참여자 상태

- [ ] 같은 방에 두 계정이 입장하면 참여자 목록에서 online 표시
- [ ] 한 계정이 탭 종료 또는 다른 방 이동 시 상태 갱신
- [ ] 현재 방 참여자 목록이 새로고침 없이 유지/변경되는지 확인

## 6. unread count

- [ ] Rooms 목록 각 방 우측 unread 배지 확인
- [ ] 채팅방 탭 총 unread 배지 확인
- [ ] 채팅방 내부 메시지 옆 unread 숫자 확인
- [ ] 상대가 읽으면 메시지 unread 숫자 감소 또는 숨김 확인

## 7. 알림 허브

- [ ] 데스크톱에서 벨 드롭다운 정상 노출
- [ ] 모바일에서 알림 전용 화면 정상 전환
- [ ] 받은 친구 요청만 처리 대상으로 노출
- [ ] 최근 알림이 허브 진입 시 자동 읽음 처리

## 8. 모바일 UI

- [ ] 하단 탭바 노출 확인
- [ ] 친구/채팅방/알림/설정 전환 확인
- [ ] 채팅 상세에서는 하단 탭바 숨김 확인
- [ ] 채팅 입력창과 하단 UI 겹침 없음

## 9. 기록할 이슈

- [ ] 새로고침해야만 반영되는 흐름
- [ ] 중복 친구 요청/중복 알림
- [ ] 1:1 방 재사용 실패
- [ ] unread count 불일치
- [ ] 참여자 수/목록 불일치
- [ ] 모바일 레이아웃 깨짐
