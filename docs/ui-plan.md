# UI 계획 (카카오톡 스타일)

기준일: 2026-03-05

## 목표

- 좌측 패널은 방 탐색(검색) 중심으로 단순화한다.
- 신규 방 생성은 우측 하단 FAB(+) + 모달로 분리한다.
- 기존 채팅 기능(입장/송수신/히스토리/참여자/오토스크롤)은 유지한다.

## 화면 구조

1. 상단 바
- 서비스명
- 연결 상태 배지 (`Connected`, `Disconnected`)
- 현재 사용자/소켓 정보

2. 좌측 패널: 방 목록
- 방 검색 입력
- 방 목록(이름/최근 메시지/시간)
- 우측 하단 FAB(+) 버튼

3. 우측 패널: 채팅 상세
- 채팅방 헤더
- 참여자 메뉴 드롭다운(우측 상단)
- 메시지 영역(스크롤)
- 입력창 + 전송 버튼(하단 고정)

4. 방 생성 모달
- 제목/설명
- 방 이름 입력
- 취소/생성 버튼
- ESC/오버레이 클릭으로 닫기

## 반응형 규칙

- Desktop (`>= 1024px`): 2열(`320px / 1fr`)
- Tablet (`768px ~ 1023px`): 2열(`280px / 1fr`)
- Mobile (`< 768px`): 단일 화면 전환
  - 기본: 방 목록
  - 방 선택 후: 채팅 상세
- 모바일에서는 FAB 하단 여백을 늘려 입력 영역과 겹침 방지

## 상태 관리

- 전역성 상태(현재 App 기준)
  - `joinedRoomId`, `isConnected`, `errorMessage`
- 목록/탐색
  - `rooms`, `searchKeyword`, `filteredRooms`
- 생성 모달
  - `isCreateRoomModalOpen`, `newRoomName`
- 채팅
  - `messages`, `participants`, `text`

## 완료 기준

- 검색/FAB/모달 동작 정상
- 방 생성 후 자동 입장 정상
- 참여자 메뉴와 기존 채팅 동작 회귀 없음
- 모바일에서 입력창/FAB 겹침 없음
