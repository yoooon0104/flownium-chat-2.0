# UI Plan (Kakao-style Pattern)

기준일: 2026-03-04

## 목표

- 현재 테스트용 단일 화면을 메신저형 2단 레이아웃으로 전환한다.
- 카카오톡과 유사한 사용 패턴을 따르되, 직접 복제는 피한다.
- 기존 기능(소켓 연결, 방 입장, 메시지 송수신, 히스토리 로드, 인증 에러 표시)은 유지한다.

## 화면 구조

1. 상단 앱 바
- 서비스명
- 연결 상태 배지 (`Connected`, `Disconnected`)
- 현재 사용자 정보(토큰 기준 닉네임, 임시)

2. 좌측 패널: 채팅방 목록
- 검색 입력
- 채팅방 리스트 아이템(이름, 최근 메시지, 시간)
- 새 채팅 버튼(초기에는 비활성 또는 임시 동작)

3. 우측 패널: 채팅 상세
- 헤더(방 이름, 참여자 요약)
- 메시지 리스트
- 입력 영역(텍스트 입력 + 전송 버튼)

## 반응형 규칙

- Desktop (`>= 1024px`): 좌우 2단 (`320px / 1fr`)
- Tablet (`768px ~ 1023px`): 좌우 2단 (`280px / 1fr`)
- Mobile (`< 768px`): 단일 화면 전환
  - 기본: 채팅방 목록
  - 방 선택 시: 채팅 상세
  - 뒤로가기 버튼으로 목록 복귀

## 컴포넌트 분리 계획

- `ChatLayout`
- `TopBar`
- `RoomListPanel`
- `RoomListItem`
- `ChatPanel`
- `MessageList`
- `MessageBubble`
- `MessageComposer`
- `ConnectionBadge`

## 상태 관리 계획

- 전역으로 올릴 후보:
  - 현재 선택 방(`activeRoomId`)
  - 연결 상태(`isConnected`)
  - 인증 에러(`authError`)
- 로컬 유지:
  - 입력값
  - 메시지 로딩 상태
  - 방 목록 검색어

## 스타일 가이드 (1차)

- 톤: 밝은 중성 배경 + 높은 대비 텍스트
- 포인트: 옐로우 계열(과도한 채도 금지)
- 메시지 버블:
  - 내 메시지: 오른쪽, 포인트 색상
  - 상대 메시지: 왼쪽, 흰/회색 계열
- 모서리/간격:
  - 카드 라운드 12~16px
  - 주요 여백 12/16/20 스케일
- 폰트:
  - 기본 시스템 산세리프 유지(한국어 가독성 우선)

## 구현 단계

1. 레이아웃 골격 전환 (`App.jsx`, `App.css`)
2. 메시지 리스트/입력 UI 컴포넌트화
3. 모바일 전환 동작 추가
4. 상태 배지/에러 배너 정리
5. 미세 스타일 조정(hover, active, skeleton)

## 완료 기준 (Definition of Done)

- 데스크톱/모바일에서 레이아웃 깨짐 없음
- 기존 기능 회귀 없음
- 인증 실패/DB 미연결 상태가 UI에서 명확히 보임
- 주요 문서(`architecture`, `wbs`) 상태 갱신
