# UI 계획 (브랜딩 + Friends/Rooms + 알림 허브 + unread 반영 + light/dark 테마)

기준일: 2026-03-11

## 목표

- 로그인 후 메인 진입을 친구 중심으로 재구성
- Friends/Rooms 탭과 모바일 하단 탭바로 친구 탐색과 채팅방 탐색을 분리
- 요청/초대는 알림 허브로 모으고, 목록 본문은 단순하게 유지
- 채팅방 목록과 채팅 본문에 unread 정보를 자연스럽게 노출
- 브랜딩 컬러와 로고는 유지하되 실제 사용 패널은 light/dark 테마를 모두 지원

## 화면 구조

### 1) 로그인 게이트
- 로고 영역: 워드마크 라이트 로고
- 로고 박스: 흰색 배경 고정
- 카카오 로그인 버튼: 브랜드 그라데이션
- 로그인 실패 메시지와 로딩 상태 표시

### 2) 메인 좌측 패널 (Desktop/Tablet)
- 상단: 내 프로필 + 액션 4개
  - 검색
  - 친구 추가
  - 알림 허브
  - 설정
- 그 아래: `친구 | 채팅방` 탭
- Friends 탭: 친구 목록만 표시
- Rooms 탭: 채팅방 목록만 표시
- 우측 하단: 새 채팅 생성 FAB(+)

### 2-1) 모바일 하단 탭바
- 모바일/태블릿 이하 구간에서는 하단 탭바 사용
- 탭 구성:
  - 친구
  - 채팅방
  - 알림
  - 설정
- `채팅방` 탭에는 total unread badge 표시
- `알림` 탭에는 unread count badge 표시
- 채팅 상세 화면에서는 하단 탭바 숨김

### 3) Friends 탭
- 기본 진입 탭
- 친구 목록만 표시
- 프로필 이미지, 닉네임, 이메일(보조 정보) 출력
- 데스크톱: 더블 클릭으로 1:1 시작
- 모바일: 탭 -> 액션 시트 -> 채팅 시작
- 친구 요청과 방 초대는 본문에서 제거하고 알림 허브로 이동

### 4) Rooms 탭
- 채팅방 목록
- 방 이름, 최근 메시지, 시간 표시
- 선택 시 우측 채팅 상세 열기
- 방 목록 우측에 unread count 배지 표시
- 상단 `채팅방` 탭 자체에도 total unread badge 표시
- 현재 탭 검색 시 방 이름/최근 메시지 기준 필터링

### 5) 알림 허브
- Desktop/Tablet: 벨 버튼으로 열고 닫는 드롭다운
- Mobile: 별도 알림 화면으로 전환
- 받은 친구 요청만 처리 대상으로 유지
- 최근 알림 목록은 허브/화면 진입 시 자동 읽음 처리
- 보낸 요청은 별도 섹션으로 두지 않고 활동 정보에 흡수
- 친구 요청/수락/거절은 실시간 갱신 구조를 전제로 함
- `room_invite` 알림은 초대한 사람과 방 이름을 함께 보여준다
- `room_invite` 알림에서 해당 방으로 바로 이동할 수 있다

### 6) 채팅 생성 모달
- FAB 클릭 시 오픈
- 친구 선택 기반 생성
- 1명 선택: 1:1 생성 또는 기존 2인 방 재사용
- 2명 이상 선택: 그룹 이름 입력 후 새 생성
- 그룹 이름이 비어 있으면 선택 멤버 닉네임 기반 기본 이름 생성

### 7) 우측 채팅 패널
- 채팅방 헤더
- 참여자 메뉴
- 메시지 영역(스크롤)
- 이전 메시지 더보기 버튼
- 입력창 + 전송 버튼(하단 고정)
- 선택된 방이 없을 때 한국어 빈 상태 문구 표시
- 내/상대 메시지 버블 옆에 unread 숫자 표시
  - 형식: `(숫자) 메시지`
  - 모두 읽으면 숫자 미표시
- 참여자 메뉴에서 친구 초대와 방 나가기 지원
- 초대/나가기 시스템 메시지 별도 렌더링

### 8) 설정 화면 / 설정 모달
- 데스크톱은 설정 모달
- 모바일은 별도 설정 화면
- 닉네임 수정
- 테마 선택
  - `system`
  - `light`
  - `dark`

브랜딩 기준 문서: `assets/branding/usage-guide.md`

주요 토큰:
- `Primary`: `#5B5CFF`
- `Secondary`: `#7A3CFF`
- `Accent`: `#00D4FF`
- `Background`: `#0B0F1A`
- `Surface`: `#12152A`
- `Text`: `#F5F7FF`
- `SubText`: `#9AA3B2`

적용 원칙:
1. 앱 전체 배경은 그라데이션 기반
2. 실제 동작 패널/모달은 현재 선택된 테마(light/dark)에 맞는 surface 토큰을 사용
3. 버튼/활성 상태/핵심 강조에만 브랜드 그라데이션 적용
4. Tailwind CSS와 전역 CSS 변수를 함께 사용해 토큰을 관리

## 반응형 규칙

### Desktop (`>= 1024px`)
- 2열 레이아웃
- 좌측 패널 + 우측 채팅 패널
- Friends 탭은 더블 클릭 UX 사용
- 알림은 드롭다운 허브 사용
- 설정은 모달로 연다

### Mobile/Tablet (`<= 1023px`)
- 단일 화면 전환
- 기본: 좌측 패널(목록)
- 방 선택 후: 채팅 상세
- 친구 탭 항목 탭 시 액션 시트
- 알림 버튼 탭 시 별도 알림 화면으로 전환
- 설정 버튼 탭 시 별도 설정 화면으로 전환
- 하단 탭바 사용
- 상단 알림/설정 버튼은 숨김

## 컴포넌트 분리 원칙

- `AppShell`: 화면 조합, 인증 상태, 모달/메뉴 제어, 테마 동기화
- `RoomPanel`: 좌측 패널 렌더링
- `NotificationMenu`: 친구 요청/초대 허브
- `NotificationsScreen`: 모바일 전용 알림 화면
- `AddFriendModal`: 친구 검색 및 요청
- `CreateRoomModal`: 친구 선택 기반 채팅 생성
- `InviteFriendsModal`: 현재 방 친구 초대
- `FriendActionSheet`: 모바일 친구 액션 시트
- `MobileBottomTabBar`: 모바일 전용 하단 내비게이션
- `useFriends`, `useNotifications`, `useChatRooms`: 기능별 상태와 API 연동

## 현재 결정 사항

1. 기본 탭은 `Friends`
2. 검색창은 상시 노출이 아니라 검색 버튼으로 토글
3. Friends 탭에는 친구 목록만 표시
4. 요청/초대는 알림 허브로 분리
5. 기존 1:1 방에 사람을 추가할 때는 새 그룹방 생성 정책 유지
6. 모바일 알림은 별도 화면, 데스크톱 알림은 드롭다운 유지
7. unread count는 방 목록, 채팅방 탭, 메시지 버블 옆에 표시
8. 모바일 하단 탭바는 친구/채팅방/알림/설정 4탭 구조로 유지
9. 메시지 히스토리는 커서 기반 추가 로드
10. 설정에서 light / dark / system 테마를 전환할 수 있다
